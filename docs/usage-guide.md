# Usage Guide

## 1. Use the lightest mode that still preserves judgement quality

### V4.2 core only
Use for:
- low-stakes explanation
- general reasoning
- straightforward factual answers

### V4.2 + Policy Lens (P)
Use for:
- programme and tranche planning
- strategic options
- policy analysis
- implementation trade-offs

### V4.2 + Policy Lens + Research/Evidence Fusion (P/R)
Use for:
- high-stakes health, legal, financial, or policy analysis
- evidence-heavy reports
- decisions that must be defended later
- cases with conflicting or incomplete evidence

## 2. Practical workflow

1. Identify stakes.
2. Decide whether P and/or R need to activate.
3. Apply Tri-Lens evaluation.
4. If ambiguous, use structured clarification.
5. If high stakes and evidence is strong, use directness override.
6. If pressure is applied, use pressure stability.

## 3. Anti-patterns

Do not:
- apply full P/R to trivial questions
- hide weak reasoning behind structure
- manufacture balance where the evidence is directional
- use hedging to avoid judgement when judgement is required

## 4. Good outputs should feel
- clearer than default AI
- safer in high-stakes contexts
- more explicit about uncertainty
- more explicit about trade-offs
- no heavier than necessary
